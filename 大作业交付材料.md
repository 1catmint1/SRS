## 1. 项目概述

### 1.1 项目背景
本项目旨在建立云南省企业就业失业数据的数字化采集和管理体系，实现企业数据的在线填报、三级审批流转(企业端-市级-省级)、数据统计分析和宏观监测。

### 1.2 核心功能
- ✅ 用户身份认证与权限管理 (JWT + RBAC)
- ✅ 企业基础信息备案与审批
- ✅ 三级审批流转机制
- ✅ 审计留痕与数据保护
- ✅ 可视化数据展示大屏
- 🔲 月度调查期填报 (开发中)
- 🔲 数据统计分析与趋势预测 (规划中)

### 1.3 技术亮点
- **微服务架构**: 基于FastAPI的RESTful API设计
- **安全认证**: JWT令牌认证 + PBKDF2密码加密
- **权限控制**: 基于角色的访问控制(RBAC)
- **审计追踪**: 完整的操作日志记录
- **可视化界面**: Streamlit构建的交互式数据大屏

## 2. 系统架构

### 2.1 技术栈
```
前端层: Streamlit (数据可视化大屏)
API层: FastAPI (RESTful接口)
认证层: JWT + OAuth2
数据层: 模拟数据库 (Mock DB)
部署层: Uvicorn (ASGI服务器)
```

### 2.2 项目结构
```
SRS/
├── main.py                 # FastAPI应用入口
├── dashboard.py           # Streamlit可视化大屏
├── api/                   # API路由模块
│   └── routers/
│       ├── auth.py        # 身份认证路由
│       └── province.py    # 省级业务路由
├── core/                  # 核心功能模块
│   ├── dependencies.py   # 依赖注入和权限检查
│   ├── security.py       # 安全认证(JWT、密码加密)
│   └── swagger_ui.py    # 自定义Swagger界面
├── db/                    # 数据库模块
│   └── mock_db.py       # 模拟数据库
└── schemas/               # 数据模型
    └── api_models.py    # API数据模型
```

### 2.3 数据库设计

#### 核心数据表
1. **t_enterprise_info** (企业信息表)
   - enterprise_id (PK): 企业编号
   - enterprise_name: 企业名称
   - filing_status: 备案状态 (0:待备案, 1:已备案, 2:已退回)

2. **t_operation_log** (操作审计日志表)
   - log_id (PK): 日志编号
   - user_id: 操作用户ID
   - operation_type: 操作类型
   - table_name: 操作表名
   - record_id: 记录ID
   - old_value: 修改前值
   - new_value: 修改后值
   - reason: 操作原因
   - operation_time: 操作时间

3. **角色权限映射表**
   - 角色ID 1: 省级管理员 (PRO_AUDIT, PRO_ADMIN)
   - 角色ID 2: 市级审核员 (CTY_AUDIT)
   - 角色ID 3: 企业用户 (ENT_SUBMIT)

## 3. API接口文档

### 3.1 认证接口

#### 用户登录
```
POST /api/v1/auth/login
Content-Type: application/x-www-form-urlencoded

Parameters:
- username: 用户名
- password: 密码

Response:
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

### 3.2 业务接口

#### 省级企业备案审批
```
POST /api/v1/province/audit-filing
Authorization: Bearer {access_token}
Content-Type: application/json

Request Body:
{
  "enterprise_id": 1001,
  "action": "APPROVE",  // 或 "REJECT"
  "reason": null  // 退回时必填
}

Response:
{
  "status": "success",
  "msg": "审批流程结束：企业【昆明市某制造企业】已被通过",
  "current_status": 1,
  "audit_log": {
    "user_id": 1,
    "operation_type": "FILING_APPROVE",
    "table_name": "t_enterprise_info",
    "record_id": 1001,
    "old_value": "0",
    "new_value": "1",
    "reason": "省级审核通过",
    "operation_time": "2026-03-30 20:15:30"
  }
}
```

### 3.3 权限说明
- **省级管理员**: 拥有企业备案审批、系统管理权限
- **市级审核员**: 拥有数据初审权限
- **企业用户**: 拥有数据填报权限

## 5. 安全机制

### 5.1 身份认证
- **JWT令牌**: 有效期120分钟
- **密码加密**: PBKDF2_SHA256算法
- **会话管理**: 基于令牌的无状态认证

### 5.2 权限控制
- **RBAC模型**: 基于角色的访问控制
- **接口级权限**: 每个API接口都有权限要求
- **越权拦截**: 自动检测并拦截越权操作

### 5.3 审计留痕
- **操作日志**: 记录所有数据修改操作
- **完整追踪**: 包含操作用户、时间、前后值、原因
- **防篡改**: 日志不可修改，确保数据真实性

- 内存: 最低2GB
- 磁盘: 最低500MB

### 9.2 部署步骤
```bash

git clone <repository_url>
cd SRS

# 2. 创建虚拟环境
python -m venv .venv
source .venv/bin/activate  # Linux/Mac

.venv\Scripts\activate  # Windows

# 3. 安装依赖

streamlit run dashboard.py
```

- **JWT密钥**: 在core/security.py中配置SECRET_KEY
